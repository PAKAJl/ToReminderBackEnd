

# ToReminder API

Бэкенд-сервис для управления задачами, построенный на современной архитектуре с использованием Node.js и облачной базы данных Turso.

## 🚀 Технологический стек

* **Runtime:** Node.js (v22+)
* **Framework:** Express 5
* **ORM:** Drizzle ORM
* **Database:** Turso (SQLite/libSQL)
* **Development:** Nodemon для автоперезагрузки кода

---

## 🛠 Структура данных (Schema)

Таблица `tasks` включает следующие поля:

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer | Первичный ключ (автоинкремент) |
| `title` | Text | Название задачи (обязательно) |
| `description`| Text | Подробное описание |
| `status` | Text | Статус задачи (по умолчанию: `todo`) |
| `priority` | Text | Приоритет (по умолчанию: `low`) |
| `date` | Text | Дата выполнения |
| `token` | Text | Идентификатор пользователя/сессии (по умолчанию: `test`) |

---

## 🚦 Эндпоинты (API Routes)

Все основные маршруты доступны по префиксу `/api/tasks`:

* **GET `/`** — Проверка работоспособности API.
* **GET `/api/tasks`** — Получение списка всех задач.
* **POST `/api/tasks`** — Создание новой задачи.
    * *Payload:* `{ "title": "String", "description": "String", "priority": "high" }`

## ⚙️ Установка и запуск

### 1. Клонирование и установка зависимостей
```bash
git clone <ваш-репозиторий>
cd ToReminderBack
npm install
```

### 2. Настройка окружения
Создайте файл `.env` в корневом каталоге и добавьте ваши данные Turso:
```env
PORT=3000
TURSO_DATABASE_URL=libsql://your-db-name.turso.io
TURSO_AUTH_TOKEN=your-auth-token
```

### 3. Работа с базой данных (Drizzle Kit)
Генерация и применение миграций:
```bash
npx drizzle-kit push
```

### 4. Запуск
* **Режим разработки (с автоперезагрузкой):**
    ```bash
    npm run dev
    ```
* **Продакшн запуск:**
    ```bash
    npm start
    ```

## 📂 Архитектура проекта
Проект следует модульной структуре для обеспечения чистоты кода:
* `src/app.js` — Инициализация Express, Middleware и маршрутов.
* `src/db/` — Конфигурация подключения к Turso и описание схем.
* `src/routes/` — Определение маршрутов приложения.
* `src/controllers/` — Бизнес-логика обработки запросов.
* `src/services/` — Прямое взаимодействие с базой данных через Drizzle.
